# Refund Request スキル

## 概要

返金・キャンセル請求メールを、状況に応じた最適な戦略で作成するスキル。
単にテンプレを埋めるのではなく、相手企業のポリシーを調査し、交渉力のある文面を構築する。

## ワークフロー

### Step 1: 状況ヒアリング

以下を確認（不明な場合はユーザーに質問）:

1. **シナリオ分類**: どの返金理由に該当するか
2. **サービス / 商品名**: 対象の企業・サービス名
3. **金額**: 請求額
4. **日付**: いつ課金 / 購入されたか
5. **Transaction ID / 注文番号**: あれば
6. **使用状況**: サービスをどの程度使ったか
7. **言語**: 英語 or 日本語（相手企業に合わせて推奨を提示）
8. **ユーザーの感情・スタンス**: 怒っている / 冷静 / 自分にも非がある等

シナリオの詳細は [references/scenarios.md](references/scenarios.md) を参照。

### Step 2: 相手企業のポリシー調査

**必ず実行する。** Web 検索で以下を調査:

1. 対象サービスの **返金ポリシー / Terms of Service / Return Policy**
2. **返金期限**（購入後何日以内か）
3. **問い合わせ先**（メールアドレス、サポートフォーム URL）
4. **他ユーザーの返金成功事例**（Trustpilot、Reddit、Twitter 等）

**推奨検索クエリパターン**:
- `"[Service Name] refund policy"` — 返金ポリシーの直接検索
- `"[Service Name] terms of service"` — 利用規約の検索
- `"[Service Name] cancel subscription refund"` — 解約・返金の具体例
- `"[Service Name] refund experience" site:reddit.com` — 他ユーザーの体験談
- `"[Service Name]" site:trustpilot.com` — レビューサイトでの評判

調査結果をもとに、ユーザーが返金ポリシーのどの条件を満たしているかを整理し、メールに活用する。

**調査後、文面作成に進む前に、以下をユーザーに共有する:**
- 見つかった返金ポリシーの要約
- ユーザーが満たしている条件
- 成功率の見積もり（[references/scenarios.md](references/scenarios.md) の成功率の目安を参照）
- 推奨する問い合わせ先（メールアドレス、フォーム URL 等）

### Step 3: オプション要素の選定

ユーザーの状況に応じて、メールに組み込むべきオプション要素を提案する。
詳細は [references/optional-elements.md](references/optional-elements.md) を参照。

主なオプション:
- 自己責任の認識（解約忘れ等の場合）
- 将来の再利用意思
- 返金手数料の負担了承
- 利用規約の引用
- 代替案の受容（クレジット、部分返金等）
- 期限内である旨の強調
- 証拠の添付（スクリーンショット等）
- 顧客歴の長さアピール（長期利用者の場合）

**ユーザーに「どのオプションを入れるか」を確認してから文面作成に進む。**

ユーザーが「おまかせ」「全部任せる」と言った場合は、[references/optional-elements.md](references/optional-elements.md) の「オプション選定のフローチャート」に従って自動選定し、選んだオプションをユーザーに報告してから次の Step に進む。

### Step 4: メール文面作成

`message_compose_v1` ツールを使い、**2パターン**を生成。`message_compose_v1` が利用できない環境の場合は、Markdown のコードブロックで 2 パターンを提示する:

| バリアント | 特徴 |
|-----------|------|
| **Polite but firm** | 丁寧だが主張が明確。情報を網羅的に提示。やりとりの往復を減らす。 |
| **More concise** | 要点のみ簡潔に。忙しいサポート担当に刺さりやすい。 |

#### メール構成の基本テンプレート

```
1. 挨拶 & 目的の明示
2. トランザクション詳細（箇条書き）
3. 状況説明 + 返金理由
4. [オプション要素をここに挿入]
5. ポリシー引用（該当する場合）
6. 自動更新停止済みの旨（該当する場合）
7. 締め & 追加情報提供の用意
8. 署名
```

#### 文面作成のルール

- **件名に Transaction ID / 注文番号を含める**（検索性向上）
- **冒頭で目的を明示**する（"I am writing to request a refund for..." / 「返金をお願いしたくご連絡しました」）
- **感情的な表現を避ける**（怒りをぶつけるのではなく、事実ベースで構築）
- **相手のポリシーを引用**して、条件を満たしていることを示す
- **自己責任部分は "regret" レベル**に留める（"take full responsibility" は言いすぎ）
- **具体的な希望を明記**する（全額返金 / 部分返金 / クレジット等）
- **レスポンス期限の目安**を暗示する（"at your earliest convenience" 等）

### Step 5: 送信前チェックリスト

メール作成後、[references/checklist.md](references/checklist.md) を参照し、**今回のケースに該当する項目のみ**を抽出してユーザーに提示する。全項目を機械的に貼り付けるのではなく、不要な項目は省くこと。

### Step 6: フォローアップ戦略の提示

[references/followup-strategy.md](references/followup-strategy.md) を参照し、以下を簡潔に提示:
- 返信がない場合のタイムライン概要（Day 3 / 7 / 14 / 21+）
- 次のフォローアップの具体的な日付（今日から計算）
- 最終手段としての chargeback 期限の注意喚起

フォローアップメールのテンプレートは、ユーザーが実際にフォローアップを依頼した時点で生成する（初回メール作成時にはタイムラインのみ提示）。

## 注意事項

- **chargeback（クレジットカード異議申し立て）は最終手段**として提示する。先に企業への直接連絡を推奨。
- 法的助言は行わない。法的手段が必要な場合は専門家への相談を推奨。
- ユーザーが明らかに不当な返金を求めている場合（使い倒した後の返金等）は、成功可能性が低いことを正直に伝える。
