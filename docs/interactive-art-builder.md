# interactive-art-builder

**ビジュアル × サウンド × 操作**のインタラクティブ・アート作品を、アイデア出しから実装・検証・公開まで一気通貫で作るワークフロースキルです。「なんか気持ちいい作品を作りたい」程度の曖昧な状態から始めても、3 ラウンドの対話ウィザードでコンセプトを固め、動く・鳴る・触れる作品に到達します。

実作品での実証を経て抽出したワークフローです（初出作品: [CatharsisField](https://catharsisfield.autumn-wave-9579.workers.dev/) ─ マウス長押しで緊張を溜め、離した瞬間に爆発させるカタルシス装置）。

## できること

- **アイデアの具体化**: 感情ゴール（カタルシス/癒し/快感連打…）→ コア操作（溜めて解放/撫でる/弾く…）→ 見た目・音・公開計画を、選択式ウィザードで段階的に確定。各質問に推奨案と「お任せ」があるので、全部お任せでも作品になります
- **2 スタック対応**:
  - **ウェブ**: p5.js + Web Audio + Strudel。URL 1 つで共有でき、スマホでも動く。環境構築が軽い（Node だけ）
  - **ネイティブ**: SuperCollider + Processing + Tidal Cycles。低レイテンシ・高音質で、展示・インスタレーション向き
- **実証済みの落とし穴を回避**: 20 項目の知識ベース（Processing の alpha レンジ、Web Audio の Karplus-Strong 制約、fps 判定の罠など）を実装時に自動参照。初見で数時間溶けるバグを踏まずに済みます
- **数値検証**: 「動いた気がする」で終わらせず、合成イベントの自動 E2E・音響振幅のサンプリング・fps 実測で裏取りしてから引き渡します
- **公開まで**: Cloudflare Workers へのデプロイ、OG 画像生成、ライセンス確認（Strudel = AGPLv3 の告知）まで面倒を見ます

## 対応／前提

| 項目 | 内容 |
|------|------|
| プラットフォーム | ウェブ（p5.js + Web Audio + Strudel）／ネイティブ macOS（SuperCollider + Processing + Tidal Cycles）／両方 |
| 入力 | マウス・タッチ（v1 のサポート範囲。マイク・カメラは実験的） |
| 外部依存 | ウェブ: Node.js + pnpm。ネイティブ: Homebrew（SuperCollider / Processing / GHC は対話中に導入手順を案内） |
| 対話ツール | AskUserQuestion 対応環境で最良。非対応環境では番号付きリストに自動フォールバック |
| ライセンス注意 | ウェブ版で Strudel（パターン音楽層）を使うと **AGPLv3**（作品ソースの公開義務）。ウィザードが公開計画の質問時に必ず告知します。パターン層なしなら制約なし |

## 使い方

新しい空のディレクトリで Claude Code を開き、作りたいものを一言伝えるだけです。ウィザードが起動し、質問に答えていくとコンセプトシート（`docs/concept.md`）が生成され、承認すると実装が始まります。

### ウィザードの流れ

1. **R1 体験の核**: 何を感じさせたいか（感情）→ どう触るか（操作。ASCII プレビュー付き）→ 参照イメージ
2. **R2 感覚の設計**: ビジュアルトーン → 色パレット（hex 提示）→ 音の性格 → 音階
3. **R3 実装と公開**: ウェブ/ネイティブ → 入力 → 公開計画 → 規模
4. コンセプトシート提示 → 承認 → 実装開始（以降は環境構築〜検証まで自律で進み、要所でだけ確認が来ます）

## プロンプトサンプル

### 1. ゼロから（アイデアなし）

```
インタラクティブアートを作りたい。アイデアはまだ無いので相談しながら決めたい。
```

→ ウィザードが R1 から順に起動します。迷ったら各質問の ⭐ 推奨か「お任せ」を選べば OK。

### 2. 雰囲気だけ決まっている

```
深海みたいな静かな作品を作りたい。触ると光る生き物が集まってくるような。
音はアンビエント系で。
```

→ 感情ゴール（癒し）・参照イメージ（深海）・音の性格が既に埋まっているので、ウィザードは残り（コア操作の詳細・パレット・プラットフォーム等）だけ質問します。

### 3. 明確なコンセプトがある（ウィザード短縮）

```
「弾けるオーブ」を作って。画面に浮遊する泡をタップで割ると、ペンタトニックの
音が鳴って粒子が飛び散る。割るたびに倍音が積み重なる。白背景に淡いパステル、
ウェブで公開まで。細部はお任せ。
```

→ ほぼ全項目が埋まっているため、コンセプトシートの確認 1 回で実装に入ります。

### 4. スマホ向けの癒し系

```
スマホで通勤中に触って癒される作品。指でなぞると水面に波紋と柔らかい音が
広がる。軽量重視で、古い端末でも動くように。URL で共有したい。
```

→ R3 の規模で「軽量」が選ばれ、粒子数・描画方式がモバイル向けに調整されます。

### 5. 展示・インスタレーション向け（ネイティブ）

```
ギャラリー展示用のインタラクティブ作品。大画面 + 良いスピーカーで鳴らすので
ネイティブ（SuperCollider + Processing）で。来場者がマウスで触れる。
音の質感を最優先したい。
```

→ ネイティブスタックで環境構築から。SuperCollider の SynthDef 設計に重心が移ります。

### 6. 既存作品の変奏（テーマ差し替え）

```
CatharsisField（溜めて解放）の構造はそのままで、テーマを「花火」に変えた
バージョンを作って。溜め＝導火線が燃える、解放＝打ち上げと開花。色は
和火っぽいオレンジ〜金。
```

→ ウィザードをスキップし、既存の設計正本を雛形にテーマ差し替えで進みます。

### 7. 途中フェーズだけ使う

```
p5.js の粒子作品が手元にあるが、音が無い。このスキルの Web Audio エンジンの
作法で音を付けて。操作は既にあるクリックとドラッグ。
```

```
作品はできてるが検証が不安。このスキルの検証ハーネス（E2E・振幅・fps）だけ
回して品質を裏取りして。
```

→ フェーズ単位の部分利用も可能です（Phase 3 の音響実装だけ、Phase 4 の検証だけ等）。

### 8. 公開・仕上げだけ

```
ローカルで動いてるウェブ作品を Cloudflare Workers で公開して。OG 画像は
一番派手な瞬間で作って。ライセンス的に問題ないかも確認して。
```

## よくある質問

**Q. 音楽やプログラミングの知識が無くても使える？**
A. はい。ウィザードは全質問に日本語の説明と推奨案が付き、音階やシンセ方式などの専門判断はスキル側が持ちます。答えるのは「どう感じたいか」だけです。

**Q. どのくらい時間がかかる？**
A. ウェブ版の最小構成（Tier 1: 視覚 + ワンショット音響）なら 1 セッション。パターン音楽層・公開まで含めたフル構成は 1〜2 セッションが目安です。

**Q. 作品の著作権・ライセンスは？**
A. 生成された作品はあなたのものです。ただしウェブ版でパターン層（Strudel）を組み込む場合のみ AGPLv3 の制約（ソース公開義務）が掛かります。ウィザードが該当時に必ず告知し、パターン層なしの構成（制約なし）も選べます。

**Q. Windows / Linux では？**
A. ウェブスタックは動くはずですが未検証です。ネイティブスタックの環境構築手順は macOS（Homebrew）前提です。

## 設計思想（カスタマイズする人向け）

- **美的判断はユーザー、技術判断は AI** ─ ウィザードで聞くのは感覚の話だけ。技術選定を質問しない
- **設計正本ファースト** ─ 実装前に architecture.md（状態機械・音視覚マッピング・契約）を固定し、視覚と音響を並行実装しても破綻しない構造を作る
- **Tier 設計** ─ コア体験は依存最小で成立させ、リッチな層はオプショナルに積む
- **数値検証** ─ 振幅・fps・パケット数で裏取りしてから人間の体感テストへ
- 質問バンクの拡張は `references/wizard.md`、落とし穴の追加は `references/pitfalls.md` へ。作品を作るたびに新しい知見を pitfalls に還流させる運用を想定しています
