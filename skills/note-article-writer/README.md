# note 記事作成スキル ─ かんたん使い方ガイド

AI（Claude）と一緒に、note.com の記事を作るための道具です。
「読者は誰か」から考え、記事の骨組みを作り、本文を書き、最後に AI っぽい不自然な表現を取り除く ─ という 4 つのステップを AI が順番に進めてくれます。

プログラミングの知識は不要です。このガイドの通りに導入して、プロンプト（AI への指示文）をコピペすれば使えます。

---

## できること

- テーマを伝えるだけで、記事を**設計から完成まで**作ってくれる
- 自分の体験談を渡すと、その体験を活かした「自分にしか書けない記事」になる
- プロフィールと過去記事を登録しておくと、**あなたの文体（語尾のクセ・リズム）を真似て**書いてくれる
- 記事全部でなく、**一部だけ**頼むこともできる（タイトル案だけ、構成だけ、書いた記事のチェックだけ）
- note の作法（タイトルは 25 文字程度・煽りタイトルは逆効果、など）を踏まえて書いてくれる

---

## 導入方法

### Claude Code（デスクトップ/ターミナル）で使う場合

この `note-article-writer` フォルダを、まるごと次の場所にコピーしてください。

```
Mac の場合:      ~/.claude/skills/note-article-writer
Windows の場合:  C:\Users\あなたのユーザー名\.claude\skills\note-article-writer
```

フォルダを置いたら準備完了です。Claude Code を開いて、下のプロンプト例をそのまま送ってください。

> うまく見つからないときは、Claude Code に「`note-article-writer` というスキルフォルダを `~/.claude/skills/` に配置したいので手伝って」と頼めば、AI が案内してくれます。

### 動作確認

導入後、Claude に次のように送ってみてください。

```
note 記事を書きたい。note-article-writer スキルは使える状態？
```

「使えます」という趣旨の返事が来れば OK です。

## 自分らしさを設定する（任意・おすすめ）

そのままでも使えますが、次の 2 つを用意すると「あなたの声」で書いてくれるようになります。どちらも 10 分あれば終わります。

### 1. プロフィールを埋める

フォルダの中の **`my-profile.md`** をメモ帳などで開いて、質問に答える形で埋めてください。
仕事・大事にしている考え方・読んでほしい相手・使いたくない言葉・書きたくない話題、などを書く欄があります。全部埋めなくて大丈夫です。

### 2. 過去に書いた記事を入れる

フォルダの中の **`samples/`** に、あなたが過去に書いた note 記事（やブログ記事）の本文を `note-1.md` `note-2.md` という名前で入れてください。3 本くらいあると、語尾のクセや文のリズムまで真似てくれるようになります。

**注意: AI に書いてもらった文章は入れないでください。** AI っぽさを消すためのスキルが、AI っぽさを学習してしまいます。

設定できているか不安なときは、Claude にこう聞けば確認してくれます。

```
my-profile.md と samples の内容、ちゃんと読めてる？
どんな書き手だと理解したか教えて。
```

## 使い方（コピペで使えるプロンプト集）

### 基本: テーマだけ渡す

```
note 記事を書いて。テーマは「朝活を3ヶ月続けた体験談」
```

これだけでも動きます。足りない情報（誰に向けて書くか、など）は AI から質問してくれるので、会話で答えていけば記事になります。

### おすすめ: 体験談を渡す

自分の体験を箇条書きで渡すと、記事の質が大きく上がります。きれいな文章でなくて大丈夫です。

```
note 記事を書いて。

テーマ: 手書きの日記をやめてスマホメモにした話
ターゲット: 日記が三日坊主で終わる人
自分の体験:
・10年間、手書きの日記に何度も挫折した
・「ちゃんと書かなきゃ」と思うほど続かなかった
・スマホに一言だけ残す方式にしたら1年続いている
・きれいに書くのをあきらめたのがポイントだった
```

### タイトル案だけほしい

```
note 記事のタイトル案を5つ出して。
テーマは「子育てしながら在宅で働く工夫」。
```

### 記事の構成（骨組み）だけほしい

```
note 記事の構成だけ考えて。まだ本文は書かないで。
テーマ: 初めての一人暮らしで買ってよかったもの
ターゲット: この春から一人暮らしを始める新社会人
```

### 自分で書いた記事をチェックしてほしい

```
この下書き、AI っぽさや読みにくいところをチェックして。
直した方がいい箇所を、理由付きで教えて。

【下書き】
（ここに自分の記事を貼り付け）
```

### 読者を一緒に考えてほしい

```
note 記事の読者分析だけお願い。
「ふるさと納税の始め方」というテーマで書きたいけど、
誰に向けて書けば読まれるか一緒に考えて。
```

## うまく使う 3 つのコツ

1. **体験を渡すほど、良い記事になる。** AI が一番書けないのは「あなたの実体験」です。失敗談・数字・そのとき思ったことをメモ書きで渡してください
2. **数字は本物だけ。** このスキルは「それっぽい数字のでっち上げ」を禁止しています。記事に数字を入れたいときは、実際の数字を伝えてください
3. **出力はドラフト（下書き）。** 公開前に必ず自分で読み直してください。特に固有名詞・数字・事実関係の最終確認は書き手の役目です

## フォルダの中身（参考）

| ファイル | 役割 |
|---|---|
| `SKILL.md` | スキルの本体。AI が読む手順書 |
| `my-profile.md` | **あなたが記入する**書き手プロフィール（任意） |
| `samples/` | **あなたの過去記事を入れる**文体学習フォルダ（任意） |
| `references/note-platform.md` | note.com の作法・仕様のまとめ |
| `references/structure.md` | タイトル・見出し・書き出しのパターン集 |
| `references/style.md` | 「AI っぽくない」文体の見本帳 |
| `references/ai-polish.md` | AI 臭を取り除くチェックリスト |
| `references/explain-hard-concepts.md` | 難しい話をやさしく伝える技法 |

あなたが触るのは `my-profile.md` と `samples/` の 2 つだけ。`references/` は書き換え不要ですが、読むと「AI がどんな基準で書いているか」がわかって面白いはずです。
