# note-article-writer

**note.com 向けの記事を、段階的なワークフローで生成する**スキルです。テーマと書き手の状況を受け取り、「分析 → 構造設計 → 本文執筆 → AI 臭排除」の 4 フェーズで、AI っぽさを抑えた対話調の記事ドラフトを作成します。

設置先に依存せず単体で動作します。書き手のプロフィールや文体素材が手元にあれば任意で取り込み、なければスキル内蔵のルールだけで完結します。

---

## 4 フェーズワークフロー

| フェーズ | 目的 | 出力 |
|---------|------|------|
| 1. 分析 | 読者の悩みを具体化し、競合との差別化軸を決める | 分析レポート |
| 2. 構造設計 | タイトル・見出し・リード文の骨格を組む | 記事骨格 |
| 3. 本文執筆 | 対話調の文体で本文を書く | 記事ドラフト |
| 4. AI 臭排除 | 機械的な表現・構造を取り除いて自然な文章に | 完成記事 |

各フェーズは単独でも呼び出せます（例: 「読者分析だけ」「この文章の AI 臭チェックだけ」）。

---

## 位置づけと配布

本スキルは**クライアント配布用の note 特化スキル**として維持しています（メンテナ自身の執筆は private リポジトリの `content-writer` が担当し、トリガー競合を避けるためローカルの symlink は張っていない）。

- **配布方法**: `skills/note-article-writer/` フォルダをそのまま（または zip で）渡す。配布先向けの導入手順とプロンプト例は同梱の `README.md`（非技術者向け）に自己完結している
- **開発環境で試す場合**: フォルダを `~/.claude/skills/note-article-writer` にコピーまたは symlink すれば発動する

スキル本体の構成は次の通りです（本ファイルはメンテナ用マニュアルとして `docs/` に、配布先向けガイドはスキルに同梱される `README.md` に分離しています）。

```
skills/note-article-writer/
├── SKILL.md                       # スキル定義（エージェントがトリガー時に読む）
├── README.md                      # 配布先向けのかんたん使い方ガイド（非技術者向け）
├── my-profile.md                  # 書き手プロフィール（配布先が記入するテンプレート）
├── samples/                       # 書き手の過去記事を入れる文体学習フォルダ
│   └── README.md                  #   入れ方と「AI 生成文は入れない」ルール
├── references/                    # フェーズ別の詳細リファレンス（progressive disclosure）
│   ├── note-platform.md           #   note.com 固有仕様（一次体験優遇・タイトル 25 文字・タグ・SEO 制約）
│   ├── structure.md               #   タイトル・見出し・リード文の設計パターン
│   ├── style.md                   #   対話調の文体（Before / After・語尾・締め方）
│   ├── ai-polish.md               #   AI 臭排除のチェックリストと言い換え表
│   └── explain-hard-concepts.md   #   難解な概念をやさしく伝える技法
└── commands/
    └── note-article.md  # スラッシュコマンド定義
```

スラッシュコマンドも使う場合は、`commands/note-article.md` を各エージェントのコマンドディレクトリ（Claude Code なら `~/.claude/commands/`）へ symlink するか配置してください。

## クイックスタート

```
# 指示に従って対話的に記事を書く
/note-article

# テーマを指定して始める
/note-article 朝活を3ヶ月続けた話
```

自然言語でも発動します。

```
note-article-writer スキルを使って記事を書いて。

- テーマ: 朝活を3ヶ月続けた体験談
- ターゲット: 朝活を始めたいけど続かない社会人
- 体験: 最初は5時起きが地獄だったが、今は自然に起きられる
```

フェーズ単位の指定もできます。

```
# 読者分析だけ
note-article-writer スキルのフェーズ1で読者分析して

# AI 臭チェックだけ
note-article-writer スキルのフェーズ4でこの文章をチェックして
```

## 使い方の流れ

1. **テーマ・ターゲット・自分の体験**の 3 点を渡します（足りなければスキル側が質問します）。
2. フェーズ 1 で読者の悩みを具体化し、差別化パターン（常識の反転 / 隠れた因果 / 抽象→具体 / 比喩の発明 / 失敗の開示）から軸を 1 つ選びます。
3. フェーズ 2 でタイトル 3 案（**25 文字程度まで**・note 公式ヘルプ準拠）・H2 見出し 3〜6 個・リード文骨子を組みます。note 固有の作法は `references/note-platform.md` を参照します。
4. フェーズ 3 で対話調（一人称「僕／私」、断定しすぎない語尾、自己ツッコミ）で本文を書きます。長さの目安はショート 1,500〜2,000 字 / ミドル 3,000〜4,000 字（デフォルト）/ ロング 5,000 字〜。**出典・実体験の裏付けのない数字は書かない**（捏造防止ルール）。
5. フェーズ 4 で禁止フレーズ（「いかがでしたか？」「それでは、早速見ていきましょう」等）の除去、コロン直後の箇条書き・機械的列挙のチェック、抽象的形容詞の具体化、裏付けのない数字の検出を行います。「〜を徹底解説」「〜完全ガイド」は禁止ではなく**乱用注意**（内容が伴うときのみ使う）です。

## 書き手のクセの反映（フェーズ 0）

執筆前に、スキル同梱の 2 つの素材を必ず確認して読み込みます（content-writer と同じ「プロフィール + 実文サンプル」方式。未設定ならスキル内蔵仕様にフォールバック）。

| 素材 | 場所 | 反映先 |
|---|---|---|
| 書き手プロフィール | `my-profile.md`（記入式テンプレート） | 切り口・想定読者・文体の好み・書きたくないことの除外 |
| 文体サンプル | `samples/note-*.md`（書き手の実文） | 語尾・リズム・一人称・導入と締めのパターン。**内蔵の文体見本（style.md）より優先** |

samples には**書き手の実文のみ**を置きます（AI 生成文を置くと AI 臭を逆学習するため禁止。samples/README.md に明記）。

さらに、設置先の環境に文体ガイド・比喩素材・体験談ストックなどのファイルがあれば追加で参照します。Obsidian Vault などにプロフィール・文体ノートを持っている人は、それらを読ませると一段パーソナライズされた出力になります。

**素材があれば「より固有性の高い記事」に、なければ「汎用テンプレートに沿った記事」になる**、という関係です。

## 関連スキルとの使い分け

| スキル | 役割 | 使いどころ |
|--------|------|-----------|
| **note-article-writer**（本スキル） | note 記事を 4 フェーズで「設計から完成まで」通す | note.com に向けた記事を、構造設計込みでゼロから書き上げたいとき |
| `content-writer` | 媒体横断（ブログ / note / X / Instagram 等）の文体・フォーマット整形 | 同じ素材を複数媒体へ展開する、媒体ごとの形式に合わせるとき |
| `article-quality-enhancer` | 記事内容の「質」を 8 テクニックで底上げ（裏テーマ・対立構造・失敗プロセス等） | 書いた記事をもっと深く・独自性のある内容にしたいとき |

本スキルは「note 向けの設計 → 執筆 → AI 臭排除」を一気通貫で担当します。文体・フォーマットの幅出しは `content-writer`、内容の深掘りは `article-quality-enhancer` と組み合わせると効果的です。

---

## 注意点

- 出力はあくまで**ドラフト**です。固有名詞・数字・体験談の事実確認、公開前の読み直しは書き手側で行ってください。
- 4 フェーズはガイドであり、短い投稿ではフェーズ 1・2 を簡略化しても構いません。
- AI 臭排除（フェーズ 4）は確率を上げる工夫であって保証ではありません。最終的な「声に出して読んで自然か」の確認は人手で行うのが前提です。
