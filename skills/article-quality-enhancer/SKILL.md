# Article Quality Enhancer

AI の記事執筆に、内容レベルでの品質向上テクニックを適用するスキル。

## ユーザー固有素材の任意参照（あれば優先）

8つのテクニックはすべて、外部素材なしで単独で動作する。ただし、ユーザーの手元に以下のような素材があれば、テーマ分析・テクニック選定の前に Read で読み込むことで、ユーザー固有の体験・語彙を引用でき、凡庸さの排除が最大化される（任意。設置先・ファイルパスには依存しない）:

- 想定読者・コア独自性・NG 表現を定義したプロフィール／執筆戦略ノート — 読者層・差別化軸の特定に使う
- 比喩・たとえ話のストック — テクニック #5（比喩フレーム）の素材源
- 失敗談・体験エピソード集 — テクニック #3（失敗プロセスの注入）の素材源
- 文体ガイド・具体的な数字や固有名詞のストック — テクニック #6（具体的な数字と固有名詞）の補強

これらの素材が手元にあるかはユーザー環境によって異なる。存在すれば引用し、なければ通常通りテクニックのみで執筆する。素材の有無にかかわらず、8つのテクニックの選定・適用ロジックは変わらない。

## 2つのモード

### 執筆モード（デフォルト）

テクニック指定付きの執筆・リライト依頼、またはスキル名指定での執筆依頼で発動する。テクニックを選定・適用しながら記事を書く。

### プロンプト提案モード

ユーザーが「プロンプトを提案して」「どういう指示を出せばいい？」等と明示的に依頼した場合のみ発動する。テクニック適用済みのプロンプト文面を提案する。

## ワークフロー（執筆モード）

### Step 1: テクニック詳細の読み込み

まず [references/techniques.md](references/techniques.md) を読み込み、8つのテクニックの詳細を把握する。

**概念伝達系の記事では追加で読み込む（分岐）**

以下のいずれかに当てはまる場合は、加えて [references/explain-hard-concepts.md](references/explain-hard-concepts.md) を読み込む:

- テーマが抽象概念・技術原理・思想・社会論を含む
- 読者が非専門家または初学者と推定される
- 「難しいことを噛み砕いて伝えたい」「嫌味なく知的に伝えたい」という依頼がある

### Step 2: テーマ分析とテクニック選定

ユーザーの依頼内容から以下を読み取る:

- **テーマ**: 何について書くか
- **記事タイプ**: ブログ / note / X スレッド / その他
- **読者層**: 誰に向けて書くか（明示されていなければ推定）
- **目的**: 情報提供 / 共感獲得 / 行動喚起 / 議論提起 など

これらをもとに、8つのテクニックから **2〜4つ** を選定する。

**ユーザーがテクニックを指定している場合はそちらを優先する。** 番号（#1〜#8）やテクニック名での指定を受け付ける。

### Step 3: 選定理由の宣言

執筆の冒頭（記事本文の前）に、以下のように簡潔に宣言する:

```
適用テクニック: #1 裏テーマ / #4 読者解像度の絞り込み / #6 具体的な数字と固有名詞
```

宣言は1〜2行で簡潔に。選定理由の説明は不要。

### Step 4: テクニックを適用しながら執筆

選定したテクニックを意識しながら記事を書く。テクニックの適用は自然に行い、「このテクニックを使っています」といったメタ的な言及は記事本文には入れない。

## テクニック選定ガイド

以下は記事タイプごとの推奨テクニック。あくまで初期値であり、テーマの性質に応じて柔軟に判断すること。

### 技術ブログ
- **必須級**: #4（読者解像度）、#6（数字・固有名詞）
- **推奨**: #3（失敗プロセス）、#7（逆説導入）
- 技術記事は具体性と実用性が命。読者がどの段階で躓いているかを絞り込み、曖昧な表現を排除する。

### ビジネス・マーケティング系ブログ
- **必須級**: #2（対立構造）、#6（数字・固有名詞）
- **推奨**: #1（裏テーマ）、#7（逆説導入）
- ビジネス読者は「結局どうすればいいか」を求めている。対立構造で選択肢を提示し、数字で裏付ける。

### note（エッセイ・コラム）
- **必須級**: #1（裏テーマ）、#8（意図的な余白）
- **推奨**: #5（比喩フレーム）、#3（失敗プロセス）
- エッセイは「考えさせる」ことが価値。裏テーマで深みを出し、余白で読後の思考を促す。

### X スレッド
- **必須級**: #7（逆説導入）、#6（数字・固有名詞）
- **推奨**: #2（対立構造）
- 冒頭の1ツイートで惹きつけなければ読まれない。逆説で引っかかりを作り、数字で具体性を担保する。

### SNS 投稿（Instagram / Facebook 等）
- **必須級**: #7（逆説導入）
- **推奨**: #6（数字・固有名詞）、#3（失敗プロセス）
- 短文なのでテクニックは1〜2つに絞る。冒頭のインパクトが最優先。

### 汎用・ジャンル不明の場合
- **基本セット**: #1（裏テーマ）、#4（読者解像度）、#6（数字・固有名詞）
- まず裏テーマで軸を作り、読者を絞り、具体性を担保する。この3つで大抵の記事は大きく改善される。

### 難解な概念の伝達（論説・概念解説系）

テーマが抽象概念・技術原理・思想・社会論を含む記事、または読者が非専門家と推定される場合に適用する。

- **必須級**: #4（読者解像度の絞り込み）、#5（比喩フレーム）
- **推奨**: #1（裏テーマ）、#8（意図的な余白）

8つのテクニックが「何を書くか」を決めるなら、`references/explain-hard-concepts.md` のメソッドは「どう読者の認知に届けるか」を設計する。両者は競合しない。Step 1 でこのメソッドを読み込んだ上で、技法 A〜D（入り口の設計・言葉の扱い・構造・リズム）を 8 つのテクニックにオーバーレイ適用する。

## テクニック同士の相性

一部のテクニックは組み合わせると相乗効果がある:

- **#1 + #5**: 裏テーマを比喩フレームで一貫して表現すると、記事全体がストーリーになる
- **#3 + #4**: 読者の状況を絞り込んだ上で、その状況特有の失敗を注入すると共感が最大化する
- **#7 + #2**: 逆説で引きつけてから対立構造で深掘りすると、読み応えのある記事になる
- **#6 + #4**: 読者の解像度を上げた上で具体的な数字を入れると、情報の実用性が極めて高くなる

一方、以下の組み合わせは注意が必要:

- **#8（余白）+ #6（数字・固有名詞）**: 情報を詰め込むテクニックと余白を作るテクニックは方向が逆。両方使う場合は、本文は #6 で密度を高め、結論部分だけ #8 で余白を作るなどバランスを取る。

## 執筆パイプラインスキルとの併用（明示指示時のみ）

本スキルは執筆依頼への自動発動はしない。執筆パイプラインスキル（例: content-writer）との併用は、ユーザーが明示的に指示した場合のみ行う。併用時の役割分担:

| 担当領域 | article-quality-enhancer | content-writer |
|---------|------------------------|----------------|
| 内容の深み・独自性 | ○（本スキル） | — |
| テクニック選定・適用 | ○（本スキル） | — |
| 文体・語尾・トーン | — | ○ |
| プラットフォーム別フォーマット | — | ○ |
| フック・導入文 | テクニックとして提案 | 具体的な文体で実装 |

併用時は、本スキルで選定したテクニックの方向性に沿って、content-writer が文体を仕上げるイメージ。

## ワークフロー（プロンプト提案モード）

「プロンプトを提案して」等の明示的な依頼があった場合のみ発動する。

1. ユーザーのテーマ・目的を確認
2. 最適なテクニック2〜4つを選定
3. テクニックが組み込まれたプロンプト文面を提案（そのまま AI に渡せる形式で）
4. 各テクニックがプロンプトのどの部分に反映されているかを簡潔に説明
