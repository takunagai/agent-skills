# note.com プラットフォームリファレンス

note.com 固有の仕様と、推薦されやすい記事の傾向。執筆判断（フェーズ1〜2）とセルフチェック（フェーズ4）で参照する。

数値・仕様は変わりうるため、断定が必要な場面では note 公式ヘルプ（help-note.com）で最新を確認する。

## プラットフォームの性格（執筆判断に効く要点）

- note の推薦が優遇するのは**人間の一次体験・経験・オリジナル作品**。AI をツールとして使い、自分の一次情報を編集・構造化したものがそれに次ぐ。新規性のない量産記事は抑制される
- 推薦されやすい: 実体験レポート、失敗談、ケーススタディ、個人的な内省・観察、引用元が明確な記事
- 抑制される: 煽り型テンプレタイトル（「〇〇な人はヤバい」「これを知らなきゃオワコン」）、他人のバズ投稿の焼き直し、偽体験や疑似科学、怒り・攻撃・晒し
- ハウツーより共感（エンパシー）型コンテンツがスキを集めやすい

## 記事の構成要素（note 固有仕様）

```
[見出し画像]
記事の「表紙」。SNS シェア時にも表示される最重要ビジュアル。
アスペクト比 2:1 推奨（X・Facebook で最適表示）。

[タイトル（25文字程度まで）]
「25文字ぐらいまでに短く」が note 公式ヘルプの推奨。
結論やオチまで書く（note 編集部発の準公式ノウハウ）。
重要キーワードは冒頭（左寄せ）に配置。
●や【】などの装飾記号は非推奨。

[リード文（冒頭の約50文字）]
SNS シェア時のプレビューテキストになる部分。
記事の価値を端的に伝える。

[目次]
note は見出しから自動生成される。
見出しだけ読んでも流れがわかる構成にする（5〜7項目が目安）。

[本文]
1見出しあたり 200〜400文字で区切る。
段落は 3〜4行以内（モバイル閲覧最適化）。
一次情報（体験・データ・具体例）を必ず含める。

[まとめ + CTA]
要点の再確認。コメント促進の問いかけ、関連記事リンクなど。
```

## ハッシュタグ

- 1記事あたり 3〜5個が目安
- 「ジャンル + 読者ニーズ + 具体キーワード」の組み合わせ（例: #ChatGPT活用、#副業初心者向け）
- 設定方法は 2 つ（本文中に「#単語」を書く / 公開設定画面で入力）。**公開設定のみを使う**のがきれい。note はタグを記事下部にチップ表示するため、本文末尾にも並べると二重表示で冗長になる
- 避ける: 抽象的すぎるタグ（#気づき、#ひとりごと）、類義語の羅列（#AI #エーアイ #人工知能）

## 文字数目安

| 記事タイプ | 文字数 | 特徴 |
|-----------|--------|------|
| 日常の気づき・日記 | 500〜1,000字 | 手軽だが検索流入は弱い |
| エンゲージメント最適 | 1,000〜1,500字 | 約3分で読了。モバイル読者に最適 |
| 一般的な実質記事 | 1,500〜3,000字 | 深さと読みやすさのバランス最良 |
| SEO 重視・有料記事 | 3,000字以上 | 検索上位を狙うならこの水準から |

## SEO と note 特有の制約

- タイトル冒頭・H2/H3 見出し・冒頭段落にターゲットキーワードを自然に配置
- E-E-A-T（経験・専門性・権威性・信頼性）を意識。特に「経験」が note の強み
- 自分の関連記事同士で内部リンクを張る
- 構造化データやカスタムメタタグの追加は不可
- 外部サイトへのリンクは nofollow 属性が付く（被リンク効果なし）
