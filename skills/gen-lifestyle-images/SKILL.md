# Lifestyle Product Image Generator

商品のライフスタイル写真を、プリセットに基づいてセミオートで一括生成するワークフロースキル。

## Prerequisites

- **gen-nanobanana-images** スキルがインストール済みであること
- **GEMINI_API_KEY** or **GOOGLE_API_KEY** 環境変数が設定済みであること
- プリセットファイル（`presets/` 配下）が対象プロジェクト用に用意されていること

## Architecture: 2-Layer Design

```
gen-lifestyle-images/
├── SKILL.md              ← Layer 1: 汎用ワークフロー（本ファイル）
├── config.json           ← デフォルト生成パラメータ
└── presets/
    └── {project}.md      ← Layer 2: プロジェクト固有プリセット
```

- **Layer 1 (SKILL.md)**: シーン設計手法、商品分析、生成実行、品質管理の汎用ワークフロー
- **Layer 2 (presets/)**: ブランドトーン、商品カタログ、シーンライブラリ、スタイリング指示などプロジェクト固有データ

## Workflow (Semi-Auto)

### Phase 1: ANALYZE — プリセット読み込みと現状把握

1. **プリセット特定**: プロジェクトのワーキングディレクトリから対応するプリセットを特定
   - `presets/` 内のプリセットファイルを読み込む
   - プリセットがない場合はユーザーに確認し、新規作成を提案
2. **商品データ読み込み**: プリセットの商品カタログから全商品・カラー・商品写真パスを取得
3. **既存画像確認**: 出力先ディレクトリの既存生成済み画像をスキャンし、各商品×カラーの生成済み枚数を把握
4. **リファレンス画像確認**: トーンリファレンスとラベルリファレンスの存在を確認

### Phase 2: PLAN — シーン・商品選定プラン作成

1. **ユーザー入力の解析**: ユーザーの指示から以下を抽出
   - 生成目的（トップページ用、商品ページ用、LP用など）
   - 希望するシーンタイプ（着用、ディテール、複数アイテムなど）
   - 商品指定（特定商品 or ランダム）
   - 枚数（1商品あたり、または合計）
   - 解像度・アスペクト比の上書き

2. **シーン選定**: プリセットのシーンライブラリから最適なシーンを選択
   - ユーザーが枚数を指定した場合、それに合わせてシーン数を決定
   - シーンタイプのバランスを確保（着用:ディテール:複数アイテム）
   - 前回生成時と重複しないシーンを優先

3. **商品×カラー選定**:
   - ユーザー指定がある場合はそれに従う
   - 「ランダム」指定の場合:
     - 生成済み画像が少ないカラーを優先
     - 商品タイプの多様性を確保（ウール/パッチワーク/アップリケ/ブーツなど）
     - 代替色がない商品は例外として含む
   - 「全商品」指定の場合: 全商品から1カラーずつ選定

4. **商品×シーン割り当て**:
   - 色味とシーンの相性を考慮（暖色→明るい背景、ダーク→木目背景など）
   - 各シーンの画像数をなるべく均等に分配
   - 割り当てマトリクスをテーブルで作成

5. **プラン提示**: AskUserQuestion で以下を提示し承認を得る
   - 選定シーン一覧
   - 商品×カラー一覧
   - 割り当てマトリクス
   - 生成パラメータ（解像度、モデル等）
   - 推定生成時間

### Phase 3: GENERATE — 画像生成実行

1. **TaskCreate** でシーンごとのタスクを作成
2. シーン順に生成を実行:
   ```
   For each scene:
     For each product assigned to scene:
       a. プロンプト構築（シーン定義 + 商品特徴 + スタイリング）
       b. 商品写真2-3枚を入力画像として選択
       c. トーンリファレンス + ラベル/タグリファレンス（インソールやタグが写る場合）を参照画像に設定
       d. gen-nanobanana-images で生成
       e. PNG → JPEG (quality=92) 変換、PNG削除
       f. 生成画像を Read で目視確認
   ```
3. 進捗報告: 25%, 50%, 75%, 100% で報告

### Phase 4: REPORT — 結果報告

1. 生成結果をシーン×商品のテーブルで一覧表示
2. ファイルパス、ファイルサイズを含む
3. リテイクが必要な画像があれば指摘

## Prompt Construction

### 6-Element Structure

プリセットのシーン定義と商品データから、以下の6要素でプロンプトを自動構築:

1. **Subject**: 商品の外観記述（色、素材、デザイン、ラベル位置）
2. **Action/Pose**: シーン定義のポーズ記述（立つ、座る、歩く）
3. **Environment**: シーン定義の背景・小物記述
4. **Composition**: シーン定義のカメラアングル・フレーミング
5. **Lighting**: シーン定義の光源・色温度
6. **Style**: "Photorealistic lifestyle photography" + シーンの雰囲気キーワード

### Styling Integration

プリセットの商品スタイリング指示を組み込む:
- パンツの色・素材
- ソックスの色・素材（or 素足）
- 商品のフォーカスポイント（アップリケ、メランジ、ニットカフなど）

### Negative Constraints

gen-nanobanana-images は既定でネガティブプロンプトを自動付加しない。プリセットの「避けるべき要素」は、プロンプト内に明示的に含める（`Avoid: ...` の一節として記述するか、叙述的・肯定形で言い換えて表現する）。

## Scene Types

### Type A: 着用ライフスタイル (Wearing Lifestyle)

人が商品を履いた状態の膝下フレーミング。生活空間に溶け込むシーン。
- 構図: 膝下フレーミング、低〜中アングル
- フォーカス: 商品 + 足元周辺の生活空間
- 小物: シーンに合った生活小物（本、マグ、植物など）

### Type B: ディテールクローズアップ (Detail Close-up)

商品のアップ目ショット。素材感・ステッチ・アップリケなどの質感を強調。
- 構図: 極浅DOF (f/1.8-2.0)、商品表面にフォーカス
- フォーカス: ウール繊維、刺繍、アップリケの立体感、ブランドラベル
- 着用: あり or なし（プリセットで定義）

### Type C: 複数アイテム (Multi-Item)

同シリーズの色違い、または異なるモデルを並べた構図。
- カラーコレクション: 同モデル2-3色を美しく配置
- モデルペア: 異なるタイプの商品を対比的に配置
- 着用ペア: 2人の足元で異なるモデルを見せる

## Generation Parameters

`config.json` のデフォルト値。ユーザー指示 > プリセット定義 > config.json の優先順で適用。

## Input Image Selection

商品写真から最適な2-3枚を自動選択:
- **正面/上面ビュー** (xxxx-1.jpg): 全体の色味・デザイン把握
- **サイドプロフィール** (xxxx-3.jpg): シルエット・ラベル位置
- **ディテール** (xxxx-5.jpg等): ステッチ・質感

## Output

- 形式: JPEG (quality=92)
- 命名規則: プリセットで定義（例: `{slug}-{color}-image-{n}.jpg`）
- 保存先: プリセットで定義（例: `public/images/image/`）

## Preset File Format

プリセットファイル (`presets/{project}.md`) の構造:

```markdown
---
name: {プロジェクト名}
project_path: {プロジェクトルートパス}
output_dir: {出力ディレクトリ}
naming: "{slug}-{color}-image-{n}.jpg"
---

# Brand Guidelines
（ブランドトーン、ターゲット、デザインコンセプト）

# Photography Style
（撮影スタイル: 光源、色温度、アングル、DOF等）

# Reference Images
（トーンリファレンス、ラベルリファレンスのパス）

# Product Catalog
（全商品×カラーのテーブル: slug, 名前, カラー, 写真パス, 特徴, スタイリング）

# Scene Library
（シーン定義: 名前, タイプ, 構図, 小物, 光源, カメラ, 相性）

# Avoid
（避けるべき要素リスト）
```
