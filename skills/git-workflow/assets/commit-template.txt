# Type 一覧:
#   feat     - 新機能追加
#   fix      - バグ修正
#   docs     - ドキュメント変更
#   style    - コードスタイル変更
#   refactor - リファクタリング
#   perf     - パフォーマンス改善
#   test     - テスト関連
#   build    - ビルド関連
#   ci       - CI/CD 関連
#   chore    - その他の変更
#
# ルール:
#   - Subject は72文字以内
#   - Subject 末尾にピリオドを付けない
#   - Body は WHY と WHAT を説明
#   - 絵文字は使用禁止
#   - 破壊的変更は ! を付けるか BREAKING CHANGE: を記載
#
# 例:
#   feat(auth): ソーシャルログイン機能を追加
#   fix(api)!: レスポンス形式を変更
