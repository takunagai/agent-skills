# 構造設計リファレンス

タイトル、見出し、リード文の設計パターン。

## タイトルパターン

### 1. 数字 + ベネフィット
```
「文章力が上がる3つの習慣」
「1日5分で変わる朝の過ごし方」
```

### 2. 意外性・逆張り
```
「なぜ僕は毎日ブログを書くのをやめたのか」
「早起きが続かないのは意志の問題じゃなかった」
```

### 3. ストーリー予告
```
「副業で月10万円稼げるようになった話」
「転職して年収が下がった話をします」
```

### 4. 共感フック
```
「何を書いていいかわからないあなたへ」
「仕事が楽しくないと感じている人に伝えたいこと」
```

### 5. How to + 具体性
```
「Notion で読書メモを管理する方法」
「Claude Code で開発効率を3倍にする使い方」
```

## 見出し構成パターン

### パターン A: 問題解決型（推奨）

```markdown
## [問題提起] 読者の悩みに共感
## [原因分析] なぜその問題が起きるか
## [解決策] このアプローチを提案
## [手順] 具体的なステップ
## [注意点] 正直に限界を伝える（省略可）
## [まとめ] 要点 + 次のアクション
```

### パターン B: 体験談型

```markdown
## [きっかけ] なぜこのテーマを語るか
## [やってみた] 具体的に何をしたか
## [結果] どうなったか
## [学び] そこから得た教訓
## [あなたへ] 読者へのメッセージ
```

### パターン C: 比較型

```markdown
## [現状] よくある方法とその問題
## [発見] 新しいアプローチ
## [比較] Before / After
## [実践] 取り入れ方
## [まとめ]
```

## リード文テンプレート

### テンプレート 1: 共感スタート

```markdown
[読者の悩みを具体的なシーンで描写]

[自分も同じ経験をしたことを示す]

[この記事で得られる成果を示唆]
```

例:
```markdown
「今日こそ書くぞ」と意気込んでも、白い画面を前に手が止まる。

僕も3ヶ月前まで、まさにそんな状態でした。

でも、あるやり方を見つけてから、週に2本のペースで書けるようになったんです。
```

### テンプレート 2: 結論先出し

```markdown
[得られる成果を先に提示]

[なぜそれが可能になったか]

[この記事で詳しく解説する旨]
```

例:
```markdown
1記事1時間で書けるようになりました。

秘訣は「構造から考える」こと。

今日はその具体的な方法を、僕の失敗談も交えてお伝えします。
```

### テンプレート 3: 疑問提起

```markdown
[常識に対する疑問を投げかける]

[自分の体験で答えを示唆]

[この記事で深掘りする旨]
```

例:
```markdown
「毎日コツコツ」が本当に正解なのか、疑問に思ったことはありませんか？

僕は「毎日」をやめたら、むしろ成果が出るようになりました。

その理由を、今日は正直にお話しします。
```
