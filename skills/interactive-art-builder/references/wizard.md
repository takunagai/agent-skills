# アイデア具体化ウィザード（Phase 0）

曖昧な「なんか作りたい」を、実装可能なコンセプトシートに変換する 3 ラウンドの対話。

## 対話プロトコル

- **第一手段: AskUserQuestion**。1 ラウンド = 1 呼び出し（最大 4 問）。推奨案は選択肢の先頭に置き末尾に「(Recommended)」。コア操作（R1-Q2）には preview で ASCII アートを付ける
- **フォールバック（AskUserQuestion 非対応環境）**: 同じ質問を下記の番号付きリスト形式でテキスト提示し、ユーザーの回答を待つ。**回答が来るまで次に進まない**

```
【R1-Q1】この作品で体感させたい「感情」は？
  1. カタルシス解放 ⭐ ─ 緊張を溜めて、一気に爆発させる快感
  2. 癒し・瞑想 ─ ゆらぎに身を委ねる静けさ
  3. 快感・連打 ─ 触るたびに小気味よい反応が返る
  4. 緊張と畏怖 ─ 巨大なもの・制御できないものへの畏れ
  5. 驚き・発見 ─ 触ると予想外の変化が起きる
  6. お任せ（Claude が選ぶ）
番号で答えてください（例: 1）
```

- **アイデア持ち込み対応**: ユーザーが最初から要素を語っていたら、その項目は質問せず復唱確認に切り替え、埋まっていない項目だけ質問する。全部埋まっていればコンセプトシート生成に直行
- 全ラウンド共通で「お任せ」を用意。お任せが選ばれた項目は、他の回答との整合で Claude が決定し、コンセプトシートに「(お任せ→提案)」と明記

## R1 ─ 体験の核（3 問）

**Q1. 感情ゴール**: カタルシス解放 / 癒し・瞑想 / 快感・連打 / 緊張と畏怖 / 驚き・発見

**Q2. コア操作（動詞）**: 感情ゴールに合う 3〜4 個を下のバンクから選んで提示。**必ず ASCII プレビュー付き**（体験のビフォー/アフターを 2 コマで）。

操作バンク（感情ゴール → 相性の良い動詞）:

| 動詞 | 相性 | プレビュー例 |
|---|---|---|
| 溜めて解放 | カタルシス◎ 連打○ | `[溜め] ●●●●●←集束 / [解放] ✦＼｜／✦ 衝撃波` |
| 撫でる・かき混ぜる | 瞑想◎ 発見○ | `≈≈≈ 〜手〜 ≈≈≈ 流体が手を避けて渦を巻く` |
| 弾く・割る | 連打◎ カタルシス○ | `○ ○ *pop* ✦ 泡を割ると倍音が積み重なる` |
| 育てる・枯らす | 瞑想◎ 発見◎ | `. → ┃ → ┣━ 触れた場所から枝が伸びる` |
| 引き寄せと反発 | 畏怖◎ 発見○ | `←●→ ●● ←● 群れが指を恐れて逃げ、離れると集まる` |
| 沈黙を破る | 畏怖◎ カタルシス○ | `━━━━ 静寂の水面 / 触れた一点から轟音の波紋` |

**Q3. 参照イメージ**（選択肢 + Other で自由記述）: 「深海」「花火」「オーロラ」「機械の墓場」等の言葉 / 作品名・URL / なし

## R2 ─ 感覚の設計（4 問）

**Q1. ビジュアルトーン**: ダーク＋ネオン ⭐ / 白＋淡色ミニマル / 有機的グラデーション / レトロ CRT・グリッチ / モノクロ＋単色差し

**Q2. カラーパレット**: Q1 の回答に応じた具体 hex 2〜3 案を提示（例: ダークネオン → a. シアン #00E5FF × マゼンタ #FF2BD6 / b. アンバー #FFB300 × ティール #00BFA5 / c. ライム #C6FF00 × バイオレット #7C4DFF）

**Q3. 音の性格**: 電子シンセ（太い・現代的）⭐ / アンビエント・ドローン（漂う）/ ノイズ・インダストリアル（ざらつき）/ アコースティック風（弦・ベル）/ チップチューン（8bit）

**Q4. 音律・スケール**: マイナーペンタトニック ⭐（外れ音が出ない・浮遊感）/ メジャーペンタ（明るい）/ ドリアン（陰影）/ リディアン（浮遊・神秘）/ 無調・ノイズ主体（音程概念なし）

## R3 ─ 実装と公開（4 問）

**Q1. プラットフォーム**: ウェブ ⭐（p5.js + Web Audio + Strudel ─ 共有しやすい・環境構築が軽い）/ ネイティブ（SuperCollider + Processing + Tidal ─ 低レイテンシ・高音質・展示向け）/ 両方（ネイティブで作り込み → ウェブ移植）

**Q2. 入力**: マウス・タッチ ⭐（v1 のサポート範囲）/ キーボード併用 / マイク・カメラ（実験的 ─ 対応コストが大きい旨を説明した上で）

**Q3. 公開計画**: ローカルで楽しむ / Cloudflare Workers で公開（+OGP 共有画像）/ 公開 + GitHub ソース公開。**ウェブ + Strudel 採用時はここで AGPLv3（ソース公開義務）を必ず告知**

**Q4. 規模・性能**: 標準 ⭐（粒子 4000・60fps 目標・モバイル対応）/ 軽量（粒子 1500・古い端末も配慮）/ リッチ（シェーダ活用・デスクトップ専用）

## 出力: コンセプトシート（docs/concept.md）

```markdown
# {{作品名候補 1}} ─ コンセプトシート
- 作品名候補: 1. {{...}} / 2. {{...}} / 3. {{...}}
- 感情ゴール: {{...}}
- コア操作: {{動詞}} ─ {{1 行の体験記述}}
- 参照イメージ: {{...}}
- ビジュアル: {{トーン}}、パレット {{hex}}
- 音: {{性格}}、{{スケール}}
- プラットフォーム: {{...}} / 入力: {{...}} / 公開: {{...}} / 規模: {{...}}
- 体験の流れ（30 秒シナリオ）: {{初見ユーザーが触れてから最初のカタルシス/発見までを 3〜5 文で}}
```

- 「体験の流れ」は回答から Claude が起草する（ここがウィザードの成果物の核。操作→反応→感情の因果を書く）
- 提示して**承認を得てから** Phase 1 へ。修正要望はシートに反映して再提示
