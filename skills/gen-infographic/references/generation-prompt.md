# 画像生成用プロンプト（骨格テンプレート）

次のテンプレートを埋め込み、画像生成機能へ渡す。角括弧のプレースホルダーは値がなければ既定値で置き換えるか削除する。

二重波括弧の差し込み口は次のように埋める:

- `{{STRUCTURE}}` — `references/structures/` から内容に合う構造ファイル1つの本文を差し込む（選び方は `structures/_index.md`）
- `{{STYLE}}` — `references/styles/` からスタイルファイル1つの本文を差し込む。未指定なら `handdrawn-marker.md`（既定）

骨格は構造にもスタイルにも依存しない共通部分（内容の優先順位・キャンバス・文字ルール・禁止事項）だけを持つ。

```text
以下の資料を理解して整理し、1枚で要点が伝わる図解画像を生成してください。

# 内容と優先順位
資料から、主題、重要な要点、関係性、流れ、比較、原因と結果、結論を抽出してください。
資料にない事実、数値、固有名詞、因果関係を追加しないでください。
情報量が多い場合は、類似項目をまとめ、重要度の低い補足を削り、主題と重要な関係性を優先してください。
文字を極端に小さくして詰め込まないでください。

# キャンバス
- 用途: [用途]
- サイズと向き: [サイズと向き]
- 1枚で完結する図解
- 紙面を真正面から見た構図
- 十分な余白
- 上部に日本語タイトル
- 重要なメッセージは視線が止まりやすい位置へ配置
- 机、手、文房具、背景風景は描かない

# 構造（情報設計）
{{STRUCTURE}}

情報ブロック数や形式を固定しないでください。上記の構造を基本としつつ、内容に最も適した形に調整してください。

# スタイル（視覚表現）
{{STYLE}}

- 表現の追加指定: [表現の調整]
- 配色の追加指定: [配色]

# 文字
画像内の文章は、必ず日本語にしてください。
英単語を残してよいのは、固有名詞、正式名称、一般的な略語、コード、翻訳すると意味が失われる技術用語だけです。
装飾目的の英語は使わないでください。
タイトル、短い見出し、簡潔な補足文を中心にしてください。
長文、意味のない文字列、重複文、判読できない文字を生成しないでください。
正確さと読みやすさを最優先してください。

固有名詞、特にアルファベットの商品名・サービス名は、文字を増減させず正確に綴ってください。
同じ文字が連続する綴り（例: freee）、長音符（ー）、難読・旧字は崩れやすいので、引用符の内側の表記を厳守してください。
【必ず一字一句そのまま表示する文字列】に記載された引用符の内側は改変せず、余計な文字を加えず、読みやすく表示してください。

# 参照画像
[参照画像と役割]

参照画像がある場合は、指定された役割に従ってください。
- スタイル参照: 線、配色、余白、強調方法だけを参考にする
- イラスト参照: 指定された人物、商品、道具、図形の外観だけを参考にする
- 構成参照: 配置と視線誘導だけを参考にする
- 内容資料: 画像内の情報を図解内容として読み取る

文章の指示と参照画像が競合する場合は、文章の指示を優先してください。

# 禁止事項（全スタイル共通）
- 情報を伝えない装飾
- 不要なロゴ、透かし、署名
- 資料にない文章、数値、固有名詞、事実
（スタイル固有の禁止事項は「スタイル」セクションの指定に従う）
```
