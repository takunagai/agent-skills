# 難解な概念をやさしく伝える

専門用語・抽象概念・技術原理を、平易な言葉のまま、嫌味なく知的に伝えるための実行ガイド。
LLM が執筆時にそのまま回せる形に再構成してある。

> 出典: 深津貴之氏の論説文を分析して抽出した技法を、一般的な執筆ルールとして再構成したもの。
> 個別の技法はすべて「核心原理」から導かれる派生にすぎない。「易しい言い換えの小技集」としてではなく、
> **読者の認知モデルに足場を架ける設計図**として読む。

## いつ使うか

次のいずれかに当てはまる記事で適用する:

- テーマに専門用語・抽象概念・技術原理・思想／社会論が含まれる
- 読者が非専門家、または初学者と想定される
- 「難しいことを噛み砕いて伝えたい」「賢く見せずに知的に伝えたい」という依頼

逆に、体験談・日常エッセイなど「概念の難しさ」が主題でない記事には無理に全適用しない。
適用するのは技法のうち効くものだけでよい。

## 核心原理：難解さは「概念」ではなく「ズレ」にある

難解さは、概念そのものの難しさではなく、**読者の頭の中の地図（認知モデル）と、書き手の地図のズレ**から生まれる。

だからやるべきは「易しく言い換える」ことではない。**一文ごとに、読者の地図に足場を架ける**ことだ。
以下の技法 A〜D は、すべてこの「ズレを埋める道具」として一本につながっている。

## 技法 A. 入り口の設計 — 既知から入り、段差を小さく登らせる

- 100%の読者が体験で知っていること（学年・身体感覚・日常）から入る
- 新概念は**既知の延長線上**に置く。「この構図を、そのまま先まで延ばしてみる」
- 一段の高さを小さくする**アナロジーの梯子**。いきなり3階に連れて行かない

## 技法 B. 言葉の扱い — 意味を固定し、誤解を先回りで潰す

- 多義語が出た瞬間に意味を釘で打つ：**「ここでいう◯◯は△△ではない。□□という意味だ」**（否定で誤解を消し、肯定で定義）
- **メタ言語の合図**を置く：「念のため言っておくと」「誤解しないでほしいのだが」「要するに」
- 読者の頭で起きそうな誤解を、起きる前に書き手が口にして潰す

## 技法 C. 構造 — 把握しやすい骨格に圧縮する

1. **二項対立に圧縮する**：複雑な論点を「A か B か」の緊張に畳む（例「透明で遅い系 vs 高速でブラックボックスな系」）
2. **問いで進める**：断定でなく「〜ではないか？」。読者を*一緒に考えるモード*に引き込み、押し付けにしない
3. **抽象と具体を往復する**：概念（高空）↔ 具体例（低空）。出した抽象は、次の段落で必ず具体に降ろす
4. **未知を既知に着地させる**：抽象論を、社会がすでに受け入れた事例（金融・自動車・医療・飛行機など）に降ろし、**未知を既知に変換**する

## 技法 D. リズム — 記憶に刻む韻律

- 短文を連打したあと、極端に短い着地文で区切る（「そういう話だ。」）
- 締めは**対句**で畳む（「A ではなく、B を」）
- 論理の文章でも、終わり方を詩に寄せると残る

## 嫌味なく知的に — 知性は語彙でなく視点で出す（最重要）

知性が嫌味になるのは、**知性の出し方を間違えている**から。難語・専門用語・引用の量で賢さを演出すると、読者は「マウントされた」と感じる。

知性の源泉は**補助線**にある。平易な言葉のまま、誰も引いていなかった「ものの見方」を一本引く。視点の転換そのものが知的で、言葉は最後まで易しい。

> **難語で知性を示す人は鼻につく。新しい視点で知性を示す人は尊敬される。**
> 知性は語彙の高さで出すのではなく、視点の高さで出す。言葉はむしろ下げる。

品を守る3つの装置:

1. **一人称＋推量**：「僕は〜と考える」「〜のではないか」。断定で殴らず、私見として差し出す
2. **読者の横に立つ**：「SF っぽく聞こえるだろうか」と読者の内心を先に代弁する。上からでなく、隣で一緒に困る
3. **誇示しない比喩**：誰でも知っているものを使う。マニアックな引用で知識量を見せつけない

## 推量表現の注意（AI 臭ルールとの両立）

本メソッドは推量を推奨するが、AI 臭として避けるべき言い回しと**似て非なるもの**なので混同しない。

- **使う（品を守る装置）**：地の文での直接的な推量「〜のではないか」「〜と考える」。書き手の私見を差し出している
- **避ける（AI 臭）**：もったいぶった丁寧形の修辞疑問「〜ではないでしょうか」「〜と言えるでしょう」。中身を伴わないぼかしになりやすい

要は、**私見を差し出す推量は可、結論をぼかす修辞疑問は不可**。

## 実行チェックリスト

### 書く前（設計）

- [ ] 読者の現在地を一行で書く（何を既に知り、何を知らないか）
- [ ] 着地の問いを決める（断定 →「〜ではないか？」に変換できるか）
- [ ] 梯子の最初の段を探す（100%の読者が体験で知る入り口）
- [ ] 二項対立を1つ立てる（この話の緊張を「A か B か」に圧縮）

### 書く時（一文ごと）

- [ ] 多義語が出たら即定義（「ここでいう◯◯は△△ではない。□□だ」）
- [ ] 誤解しそうな所で先回り（「念のため」「誤解しないでほしいが」）
- [ ] 抽象を出したら次の段落で具体へ降ろす（往復）
- [ ] 知性は補助線で出す（難語に逃げず、新しい視点を1本）
- [ ] 断定したくなったら推量に変える（「である」→「のではないか」）
- [ ] 一段落＝一論点（主張が2つなら段落を割る）
- [ ] 読者の内心を代弁する一文を時々挟む

### 書いた後（推敲）

- [ ] 未知を既知に着地させたか（社会が受け入れ済みの事例へ）
- [ ] キーワードを1語に絞り反復したか（迷子防止の旗）
- [ ] 締めを対句／短文にできるか
- [ ] マウント検査（知識量を見せている箇所 → 視点に置き換える）
- [ ] 音読（苦しい長文は割る。短い着地文を1つ作る）

## 合言葉

> 既知から入り、言葉を釘で打ち、抽象と具体を往復し、問いで進め、補助線で知性を出す。
