# AI 臭排除リファレンス

AI 生成文章から「機械っぽさ」を取り除くチェックリストと修正パターン。

## 構造的な AI 臭

### コロン直後の箇条書き

**NG**:
```
ポイントは3つあります：
- ポイント1
- ポイント2
- ポイント3
```

**OK**:
```
ポイントは3つあります。

まず、〜。次に、〜。そして、〜。
```

または間に一文入れる:
```
ポイントは3つあります。順番に説明しますね。

- ポイント1
- ポイント2
- ポイント3
```

### 機械的な列挙

**NG**:
```
まず、〇〇します。
次に、△△します。
そして、□□します。
最後に、◇◇します。
```

**OK**:
```
最初にやるのは〇〇。
△△も忘れずに。
ここで□□するのがコツです。
仕上げに◇◇して完成。
```

### 同じパターンの繰り返し

**NG**（全ての段落が同じ構造）:
```
〇〇は重要です。なぜなら△△だからです。
□□も重要です。なぜなら◇◇だからです。
××も重要です。なぜなら☆☆だからです。
```

**OK**（バリエーションをつける）:
```
〇〇、これが一番大事。

□□も見逃せません。◇◇という理由があるんです。

あと、意外と知られていないのが××。
```

## 表現的な AI 臭

### 言い換え必須リスト

| AI っぽい表現 | 自然な表現 |
|-------------|----------|
| 非常に重要です | 大事なんです / ここがポイント |
| 極めて有効です | めちゃくちゃ効く / 本当に使える |
| 〜と言えるでしょう | 〜だと思う / 〜な気がする |
| 〜する必要があります | 〜した方がいい / 〜しないと困る |
| 〜を活用する | 〜を使う |
| 〜を実施する | 〜をやる / 〜する |
| 様々な | いろんな / いくつかの |
| 〜において | 〜で / 〜では |
| 〜という点で | 〜だから / 〜なので |
| しかしながら | でも / ただ |
| それにもかかわらず | それでも / なのに |
| 〜することができます | 〜できます |
| 〜していただければ幸いです | 〜してもらえると嬉しい |

### 絶対禁止フレーズ

即座に削除または書き換え:

```
いかがでしたか？
いかがだったでしょうか？
〜について詳しく解説します
〜を徹底解説
〜完全ガイド
さあ、〜しましょう！
〜してみてはいかがでしょうか
それでは、早速見ていきましょう
```

## 内容的な AI 臭

### 具体性の欠如

**NG**:
```
コミュニケーションが大切です。
相手の気持ちを考えることが重要です。
```

**OK**:
```
「ありがとう」を言うタイミングを意識してみてください。

僕の場合、朝のあいさつに「昨日の〇〇、助かりました」を
一言添えるようにしたら、チームの雰囲気が変わりました。
```

### 逆接がない

**NG**（全てが順調に進む）:
```
〇〇をしました。うまくいきました。
次に△△をしました。これも成功しました。
```

**OK**:
```
〇〇をしました。最初はうまくいった。

でも、2週間後に壁にぶつかったんです。

そこで△△に切り替えてみたら...
```

## チェックリスト

記事完成前に確認:

### 構造
- [ ] コロン直後の箇条書きがない
- [ ] 「まず、次に、最後に」の機械的列挙がない
- [ ] 段落の長さにバリエーションがある
- [ ] 同じパターンの文頭が3回以上続いていない

### 表現
- [ ] 禁止フレーズが含まれていない
- [ ] 同じフレーズが3回以上繰り返されていない
- [ ] 「非常に」「極めて」などの抽象的形容詞がない
- [ ] 冗長な接続詞（「しかしながら」など）がない

### 内容
- [ ] 各セクションに具体例やエピソードがある
- [ ] 「とはいえ」「ただ」などの逆接が適度にある
- [ ] 読者への問いかけが自然（連発していない）
- [ ] 締めが「いかがでしたか」になっていない

### 最終チェック
- [ ] 声に出して読んで自然か
- [ ] 「これ AI っぽいな」と感じる箇所がないか
